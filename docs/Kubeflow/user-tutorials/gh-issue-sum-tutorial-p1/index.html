GitHub Issue Summarization Example. Part 1

Training Keras Model with Jupyter#

Note: The following tutorial is based on the example here.

Upload the kubeflow-tutorial.zip if you did not do that before.

  1. Open Kubeflow Dashboard and click on Notebook Servers item in the left menu.
  1. Click on New Server button.
  1. Type in notebook server‚Äôs name.

  1. Choose Custom and set image:

    gcr.io/mapr-252711/issue-summarization-notebook-cpu:latest

In air gap: <registry_ip>:<port>/issue-summarization-notebook-cpu:latest

  1. Leave Workspace Volume as is. Specify at least 16 Gi of memory. Add New Data Volume, select ReadWriteMany mode and change Mount Point to a shorter name /data:
  1. Launch the notebook server and wait until its running state.

Note: In case your environment has proxy values, you need to create and apply an egress.yaml file that will allow the newly created notebook to connect to the internet. An example is shown below:

Then apply the yaml file with

$ kubectl apply -f egress.yaml

Double check that the service entry was created properly by running the command:

$ kubectl get ServiceEntry

  1. Connect to the notebook and open a new terminal. In the terminal clone repository with the notebook:

git clone https://github.com/mapr/kubeflow-examples.git

Note: If your environment has proxy values, export proxy host before cloning the repo:

$ export http_proxy=<http_proxy_host>
$ export https_proxy=<https_proxy_host>

  1. Go back to the folder list and open:

    kubeflow-examples/github_issue_summarization/notebooks/Training.ipynb

  1. In step ‚ÄúSet path for data dir‚Äù change data dir:

    %env DATA_DIR=/data

  1. In ‚ÄúPre-Process Data For Deep Learning‚Äù section comment magic function %%time (known issue for the example).

Note: In case of using proxy add environment variable in step 4 (‚ÄúDownload the github-issues.zip training data to /mnt/github-issues-data‚Äù):

%env https_proxy=<https_proxy_host>

  1. Select Cell > Run All to run all the steps or click Run button to run the notebook step by step manually.
  1. After training completes copy result files to HDFS. In the notebook‚Äôs terminal:
cd kubeflow-examples/github_issue_summarization/notebooks/
cp *.h5 /data/
cp *.dpkl /data/

The following result files will be stored in HDFS:

  • seq2seq_model_tutorial.h5 - the keras model

  • body_pp.dpkl- the serialized body preprocessor

  • title_pp.dpkl- the serialized title preprocessor
