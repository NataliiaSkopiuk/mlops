Kubeflow Pipelines (KFP)

Running a basic Pipeline#

  1. Open Kubeflow dashboard.
  1. Go to Pipelines page.
  1. Click the name of the sample, [Tutorial] DSL - Control structures.
  1. Click Create experiment and follow the prompts.
  1. Create a run by clicking on the Start button.
  1. Click the name of the run on the Experiments page.
  1. Explore the graph and other aspects of your run by clicking on the components of the graph and the other UI elements.

Running Pipeline in Jupyter Notebook#

Example from here.

  1. Create Jupyter notebook server.
  1. Connect to the notebook and click to open a New Terminal.

  1. In the terminal clone kubeflow/pipelines repository:

    $ git clone https://github.com/kubeflow/pipelines.git

Note: If using proxy, first run the bellow command:

$ export https_proxy=<https_proxy_host>

  1. Go back to the Files tab and open a notebook:

    pipelines/samples/core/lightweight_component/lightweight_component.ipynb

  1. Run the notebook step by step.
  1. In the last step click on the links provided and check respective experiment and run are available in Pipelines UI.

"Experiment link here" opens the experiment created in the Pipelines UI.

"Run link here" opens the run created in the Pipelines UI.

  1. Open Experiments page in the Pipelines dashboard. In All Experiments tab expand Default group and view pipeline graph and details per step by clicking on [View pipeline] link.
  1. In All runs tab click on the name of the run to view Graph, Run output and Config tabs.

Run Spark as a part of KFP#

Upload the kubeflow-tutorial.zip if you have not done that before.

  1. In Kubeflow UI create a new Notebook Server.
  1. Any of the default Jupyter images can be used. No specific properties or options are required.

  1. Connect to the created Notebook Server and open Python3 Notebook.

  2. In the created Notebook prepare the code for running Pipelines:

    import kfp.dsl as dsl
    import yaml
    @dsl.pipeline(
    name = "Spark pipeline demo",
    description = "Use saprk operator inside KF",
    )
    def spark_pi_pipeline():
    spark_yaml = yaml.safe_load(open("/home/jovyan/pyspark.yaml"))
    deploy_step = dsl.ResourceOp(
    name="spark-job",
    k8s_resource=spark_yaml,
    attribute_outputs={"name": "{.metadata.name}"})
    import kfp
    kfp.Client().create_run_from_pipeline_func(spark_pi_pipeline, arguments={})
  1. In the root folder of the Jupyter notebook upload .yaml file with the code for spark job (pyspark.yaml).
  1. Create new notebook (PySpark, Python 3, etc.). Upload example code for Wordcount job (spark-wc.yaml).

Note: change namespace name, path to the file input.txt in maprfs (with any content) and spark.mapr.user.secret name to match existing values.

  1. Login to the ECP Control Pane, open terminal and execute script to generate maprticket:

    $ kubectl exec -it tenantcli-0 -- su - <ldap_user_name>

Note: Use the name of the created user secret in spark job code in previous step (example: mapr-user-secret-spark-example).
